734  //
736  //
886  //
887  // A task is created by inheriting
888  // from task and providing a main() function. Initialisation of the task,
889  // including creating its waitables, should be done in the constructor.
890  // Don't forget to call the constructor of the task class!
891  //
892  // The main() is the body of the task. It should never terminate.
893  //
894  // Each task has a unique priority (an unsigned integer).
895  // A lower value indicates a higher priority.
896  // The rtos scheduler will always run the task with the higest-priority
897  // runnable (neither blocked nor suspended) task.
898  // A task runs until it changes this 'situation' by using an rtos
899  // call that changes its own state to not runnable, or the state of a
900  // higher priority task to runnable.
901  //
902  // Timers are served only when the rtos is activated
903  // by calling any of its state-changing interfaces.
904  // Hence the longest run time between such calls determines the granularity
905  // (time wise responsiveness) of the application.
906  // Within a time consuming computation a task can call release() to have
907  // the rtos serve the timers.
908  //
